Endpoint

Le point d'accès "pluriel" sera /aliments, et le point d'accès "singulier" sera /aliments/id.

Ces endpoints seront précédés par : https://eden.imt-lille-douai.fr/~prenom.nom/projet/


Actions possibles pour l'utilisateur

Liste des actions possibles
Action Verbe HTTP Endpoint Description correspondante
Lire GET /aliments Donne la liste des aliments
Créer POST /aliments Crée un aliment
Lire GET /aliments/id Affiche les détails de l'aliment correspondant à l'id
Modifier PUT /aliments/id Modifie l'aliment correpondant à l'id
Supprimer DELETE /aliments/id Supprime l'aliment correspondant à l'id

Effectuer une recherche

Les paramètres de recherches qui constituent une requête seront par exemple un aliment (l'utilisateur peut rechercher quels aliments sont composés d'un sous aliment par exemple), une date d'ajout dans la base, un taux de sel ou de graisses pour 100g inférieur à un seuil,...


Afficher un aliment

Par exemple, pour afficher un aliment avec l'id 8 : https://api.imm.com/v1/aliments/8
